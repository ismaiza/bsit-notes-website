System Analysis and Design (SAD)

1. Introduction to System Analysis and Design

System Analysis and Design is the process of studying an existing system, identifying problems, and designing a new or improved system to meet user requirements.

2. System

A system is a collection of components that work together to achieve a common goal.

3. Types of Systems

  • Physical System
  • Abstract System
  • Open System
  • Closed System
  • Manual System
  • Computer-Based System

4. System Analyst

A System Analyst is a professional who analyzes user requirements and designs information systems.

5. Roles of System Analyst

  • Requirement gathering
  • System design
  • Feasibility study
  • Communication with stakeholders
  • System documentation

6. System Development Life Cycle (SDLC)

The SDLC defines the stages involved in system development.

  1. Planning
  2. System Analysis
  3. System Design
  4. Implementation
  5. Testing
  6. Deployment
  7. Maintenance

7. Feasibility Study

  • Technical Feasibility
  • Economic Feasibility
  • Operational Feasibility
  • Schedule Feasibility

8. Requirement Analysis

  • Functional Requirements
  • Non-Functional Requirements
  • User Requirements
  • System Requirements

9. Data Collection Methods

  • Interviews
  • Questionnaires
  • Observation
  • Document review

10. System Design

  • Logical Design: What the system should do
  • Physical Design: How the system will be implemented

11. Design Tools

  • Data Flow Diagram (DFD)
  • Entity Relationship Diagram (ERD)
  • Flowcharts
  • Use Case Diagrams

12. Data Flow Diagram (DFD)

  • Process
  • Data Flow
  • Data Store
  • External Entity

13. Testing

  • Unit Testing
  • Integration Testing
  • System Testing
  • User Acceptance Testing (UAT)

14. Implementation

  • Direct Changeover
  • Parallel Changeover
  • Pilot Changeover
  • Phased Changeover

15. Maintenance

  • Corrective Maintenance
  • Adaptive Maintenance
  • Perfective Maintenance
  • Preventive Maintenance

16. Advantages of System Analysis and Design

  • Improves system quality
  • Reduces development cost
  • Enhances user satisfaction
  • Ensures proper documentation

17. Exam Tips

  • Memorize SDLC stages
  • Understand feasibility types
  • Know DFD symbols
  • Practice short notes
