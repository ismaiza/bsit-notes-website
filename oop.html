Object-Oriented Programming (OOP)

Introduction

Object-Oriented Programming is a programming paradigm based on the concept of objects. Objects contain data and methods that operate on the data.

Core Principles of OOP

1. Encapsulation

Encapsulation is the bundling of data and methods that operate on that data into a single unit called a class.

2. Abstraction

Abstraction hides implementation details and shows only essential features.

3. Inheritance

Inheritance allows a class to acquire properties and methods of another class.

a. Types of Inheritance

single inheritance,multiple inheritance,hybrid inheritance,

4. Polymorphism

Polymorphism allows one interface to be used for different data types.

a. Static polymorphism

: its achieve through method overloading and operator overloading method overloading involves defining multiple methods with the same name but different parameter lists within the same class

b. Dynamic Polymorphism

: also know as runtime polymorphism is achieve through method overiding and interface implementation.

3. Advantages of OOP

OOP improves code reusability, security, and maintainability. It makes large software systems easier to manage.

4. OOP Concepts Example

A class is a blueprint, while an object is an instance of a class. For example, a "Car" is a class, and "Toyota" is an object.
